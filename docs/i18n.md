# Internationalization (i18n)

- Library: next-intl v3 with plugin in next.config.ts.
- Locales: defined in `src/lib/i18n/config.ts` with default `ng`.
- Messages: JSON files in `src/lib/i18n/messages/*`.
- Server config: `src/i18n.ts` validates locale and loads messages.
- Middleware enforces locale prefix and redirects to default if missing.
- Use localized navigation helpers from `src/lib/i18n/navigation.ts` instead of Next.js defaults.

## Usage

```tsx
import { Link, useRouter } from "@/lib/i18n/navigation";

<Link href="/about">About</Link>
```

Switching locale: wrap paths using `getLocalizedPath`.
